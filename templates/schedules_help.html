Как работают расписания в Marshrutka

Queue-driven scheduling: расписания запускают не жёсткий payload, а первый элемент из очереди проекта.
Что такое расписание в Marshrutka?

Расписание (schedule) — это cron-правило, которое автоматически запускает контентный pipeline. В отличие от традиционных систем, где расписание привязано к фиксированному набору входных данных, в Marshrutka расписание работает через очередь проекта.

Как это работает
1
Создайте элементы в очереди проекта со статусом «В очереди».
2
Создайте расписание для этого проекта и укажите cron-выражение.
3
При каждом срабатывании расписания Marshrutka берёт первый элемент из очереди со статусом «В очереди», сортируя по позиции.
4
Элемент запускается: создаётся локальный Run, отправляется remote job, элемент помечается как «Запущен».
5
В элемент очереди записываются last_run_id, last_remote_job_id и last_launch_at.
Что происходит, если очередь пуста?

Если в очереди проекта нет элементов со статусом «В очереди», расписание срабатывает, но ничего не запускается. В логах Marshrutka появляется запись [scheduler] queue is empty, no-op. Расписание остаётся здоровым (healthy) и продолжит срабатывать по cron.

Что происходит, если первый элемент очереди невалиден?

Если у первого элемента очереди пустой payload, он переводится в статус «Ошибка» с сообщением Empty payload — cannot launch. Расписание НЕ пытается запустить следующий элемент в той же итерации.

Если remote API возвращает ошибку или сервер проекта недоступен, элемент также переводится в «Ошибка» с текстом ошибки. Исправьте элемент вручную или удалите его, чтобы расписание перешло к следующему.

Статусы элементов очереди
Черновик Не готов к запуску. Не участвует в обработке расписанием. В очереди Готов к запуску. Расписание берёт первый такой элемент. Запускается Идёт отправка на сервер проекта. Запущен Успешно отправлен на сервер. Есть run_id и remote_job_id. Ошибка Ошибка при запуске. Проверьте last_error. Приостановлен Временно исключён из очереди. Можно вернуть обратно. Архив Завершённый или ненужный элемент. Не участвует в обработке.
Как проверить, что расписание активно?
  1. Откройте страницу проекта. В блоке «Расписания» статус должен быть активно.
  2. Убедитесь, что в очереди есть элементы со статусом «В очереди».
  3. Проверьте, что проект активен (не отключён).
  4. На странице Расписания виден список всех расписаний с последним запуском.
Как безопасно тестировать расписание?
  1. Создайте элемент очереди со статусом «Черновик» — он не будет запущен расписанием.
  2. Переведите элемент в «В очереди» и нажмите «Запустить» вручную — это имитирует срабатывание расписания.
  3. Проверьте результат на странице run details.
  4. Только после успешного ручного теста включайте cron.
  5. Начните с редкого интервала (например, раз в день), чтобы избежать неожиданных запусков.
