1. Панель метрик и визуальные карточки проектов

Дашборд Marshrutka показывает карточки проектов, но не имеет KPI-метрик (всего проектов, активных запусков, успешных/упавших). Карточки состоят только из текста — нет статус-баров, иконок, прогресса.

Directional · 8/16 references

«Метрическая панель в верхней части дашборда» — встречается в 8 из 16 просмотренных SaaS-референсов (Geckoboard, Userlane, Intercom, Rippling, Chameleon, WalkMe и др.).

Как внедрить:

  • Добавить ряд KPI-метрик над карточками: «Проектов: 5», «Запусков сегодня: 3», «Успешных: 87%», «В очереди: 2»
  • Каждому проекту — визуальный индикатор статуса (progress bar выполнения последнего запуска, цветная полоса статуса)
  • Карточки проектов: добавить иконку/эмодзи типа проекта, последнее время запуска, статус расписания
Не делать еслиприложение используется как headless API и дашборд — только для отладки; тогда просто убери его.

2. Улучшение AI-панели «Маршал»

AI-панель справа тёмная, но ей не хватает полноценного UX чата: нет аватаров, состояния набора сообщения («печатает…»), не отображается время сообщений, нет сворачивания сообщений с кодом.

Directional · 6/16 references

Современные AI-chat интерфейсы (Intercom, Microsoft Copilot, Gemini) добавляют визуальную иерархию сообщений и быстрые действия.

Как внедрить:

  • Добавить аватар «Маршала» (иконка/логотип Marshrutka) слева от его сообщений
  • Индикатор «печатает…» при обработке запроса
  • Группировка шагов (инструменты → ответ) — уже есть в `agent-chat.js`, но не визуализирована в виде карточек
  • Возможность свернуть длинные сообщения с кодом/JSON (toggle)
  • Быстрые чипсы категорий: «Запустить проект», «Проверить логи», «Статус очереди»
Не делать еслитекущая функциональность AI-панели устраивает всех пользователей; опрос покажет приоритет.

3. Навигация и каркас

Текущий сайдбар содержит 4 пункта без группировки. Нет поиска, нет хлебных крошек на всех страницах, нет быстрого переключения между проектами.

Directional · 5/16 references

Конкретные изменения:

  • Разделить сайдбар на секции: «Основное» (Дашборд, Проекты) и «Интеграции» (Расписания, Публикация)
  • Добавить строку глобального поиска в сайдбар или в верхнюю панель
  • Улучшить хлебные крошки: показывать на всех страницах, добавить имя проекта на странице деталей
  • Сделать футер сайдбара менее заметным (уменьшить размер шрифта)
Не делать еслиу приложения 1–2 активных пользователя, которым текущая навигация привычна.
