# Marshrutka — руководство по настройке публикации

> Актуально на 2026-06-10.

---

## 1. Что такое publish profile

Publish profile — это конфигурация платформы для публикации видео из Marshrutka.

Профиль хранит:
- название и платформу
- учётные данные (client_id, client_secret, токены)
- привязку к конкретному каналу
- шаблоны заголовка, описания, тегов
- видимость по умолчанию (public / unlisted / private)

Профили хранятся в `data/publish_profiles.json`.

---

## 2. Как создать YouTube profile

1. Откройте `/publish/profiles/new`
2. Выберите платформу **YouTube**
3. Заполните название (например `YouTube — Зад Пегаса`)
4. В поле **Credentials JSON** вставьте:

```json
{"client_id": "xxx.apps.googleusercontent.com", "client_secret": "GOCSPX-..."}
```

5. Укажите видимость по умолчанию (для «Сапоги-сандалики» — **Приватный**)
6. Сохраните профиль

### Как получить client_id и client_secret

1. [Google Cloud Console](https://console.cloud.google.com/) → создайте проект
2. **APIs & Services** → **Library** → включите **YouTube Data API v3**
3. **OAuth consent screen** → **External** → заполните поля, добавьте свой email в Test users
4. **Credentials** → **Create Credentials** → **OAuth client ID** → тип **Web application**
5. Authorized redirect URIs: `http://localhost:9090/publish/youtube/callback`
6. Сохраните, откройте скачанный JSON — оттуда нужны поля `client_id` и `client_secret`

## 3. Как пройти OAuth

1. Откройте профиль на редактирование (`/publish/profiles/{id}/edit`)
2. В блоке **Учётные данные** нажмите **🔑 Авторизовать YouTube**
3. Подтвердите доступ в Google
4. Marshrutka получит токены и список каналов
5. Если у аккаунта один канал — профиль готов
6. Если несколько — выберите нужный канал на странице выбора

После успешной авторизации:
- `channel_id` и `channel_title` заполняются автоматически
- статус профиля: **✓ готов к публикации**
- в credentials сохраняются `refresh_token`, `access_token`, `expires_at`

---

## 4. Как привязать profile к проекту

1. Откройте `/projects/{project_id}/edit`
2. В разделе **Publish profiles** выберите профиль из списка
3. При необходимости установите его как профиль по умолчанию
4. Сохраните проект

---

## 5. Как опубликовать run

1. Дождитесь успешного завершения pipeline (статус **Успех**)
2. На странице run деталей откройте блок **Публикация**
3. Выберите профиль, заполните заголовок и описание
4. Нажмите **📤 Создать запрос** → появится строка с кнопкой **▶ Выполнить**
5. Нажмите **▶ Выполнить** — кнопка заблокируется и покажет спиннер на время загрузки

Видео загрузится в YouTube через resumable upload. В результате появится ссылка на `youtu.be/{video_id}`.

Если публикация не удалась — рядом с запросом появится пометка **⚠ ошибка**. Нажмите **подробнее** чтобы увидеть текст ошибки от YouTube API.

## 6. Как настроить 3 канала на одной Google-учётке

Marshrutka всегда использует `prompt=consent` + `access_type=offline`. Google выдаёт **новый refresh_token** при каждом подтверждении. Это позволяет:

1. **Создать профиль «YouTube — Зад Пегаса»** → авторизовать → выбрать канал 1
2. **Создать профиль «YouTube — Ежу Понятно»** → авторизовать → выбрать канал 2
3. **Создать профиль «YouTube — Сапоги-сандалики»** → авторизовать → выбрать канал 3
4. У профиля «Сапоги-сандалики» установить видимость **Приватный**

Каждый профиль хранит свой `refresh_token` и `selected_channel_id`. Три профиля = три независимых OAuth-сессии.

---

## 7. Как добавить 4-й канал потом

Без изменения кода:

1. `/publish/profiles/new` → создать YouTube-профиль
2. Ввести `client_id` / `client_secret`
3. **🔑 Авторизовать YouTube**
4. Выбрать канал
5. Привязать к проекту

Всё. Профиль готов к публикации.

---

## 8. Статусы профиля

На странице `/publish/profiles` отображается статус:

| Статус | Описание |
|--------|----------|
| ✓ готов к публикации | OAuth пройден, канал выбран, можно публиковать |
| требуется OAuth | client_id/client_secret есть, но OAuth не пройден |
| выберите канал | OAuth пройден, но канал не выбран (несколько каналов) |
| требует настройки | Не хватает учётных данных |
| отключён | Профиль выключен |

## 9. Known limitations

- **Instagram, VK, Rutube** — UI есть, адаптеров нет. Публикация работает только для YouTube.
- **Refresh token** — может истечь, если не использовался > 6 месяцев. Тогда нужно заново пройти OAuth.
- **Один refresh token на один профиль** — если переавторизовать профиль, старый refresh_token перестаёт работать.
- **YouTube Data API квота** — 10 000 единиц/день. Одна загрузка ~1600 единиц. ~6 загрузок в день.
- **Размер видео** — до 256 ГБ, до 12 часов.
